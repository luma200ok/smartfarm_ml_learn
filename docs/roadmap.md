# 🌱 스마트팜 재배 도우미 로드맵 (ML → DL → LLM)

> **목표:** 스마트팜 환경 데이터 + 잎 사진을 학습해서,
> "💧 토양 건조하니 관수, 🔬 잎에 탄저병 의심, 🌡️ 곧 고온이니 환기" 처방해주는 AI 만들기
>
> **전략:** 하나의 도메인(스마트팜)으로 ML→DL→LLM을 점진적으로 쌓되,
> **단계마다 새 데이터 모달리티(정형→이미지→언어)** 를 도입해 점프를 진짜로 만든다.

---

## 🔑 왜 이 도메인인가 — "승리 공식"

```
✅ 정형 센서 → ML 분류/예측
✅ 잎 이미지 → DL(CNN) 진단  ← ML로 불가능, DL 명분 100%
✅ 진단+환경 → LLM 자연어 처방 + 알림으로 마무리 딱
✅ 멀티모달이라 "DL이 ML 재탕" 함정이 없음 (출근비서 대비 핵심 우위)
```

---

## 🏗️ 전체 아키텍처

```
[Phase 1: ML]  센서/토양 CSV → "작물 적합도 / 환경 이상" 분류
                    ↓ 새 모달리티 추가
[Phase 2: DL]  ① LSTM → 환경 시계열 예측
               ② CNN  → 잎 사진 병해충 진단  ⭐ 차별화 핵심
                    ↓ 예측·진단 결과를 가져다 씀
[Phase 3: LLM] DL진단(병) + 환경예측(센서) + 재배가이드(RAG)
                → LLM이 자연어 처방 생성
                → 📱 이상 시 알림
   "🔬 잎곰팡이병 의심(87%). 감염 잎 제거하고 습도를 낮추세요.
    💧 토양수분 30%로 낮음 — 관수 필요. 🌡️ 2시간 뒤 32도 예측, 환기 준비."
```

## 🧭 전체 청크 지도 (끝까지 — 안 헷갈리게)

> **Phase = 큰 흐름(ML→DL→LLM), 청크 = 공부하기 좋게 쪼갠 1회분.** 청크 번호·개수는 **예상치**(진입 시 조정).
> 표기: ✅완료 · ▶다음 · ⬜예정. 시간순 상세는 [_local/progress_log.md], 개념은 [_local/concepts.md].

### Phase 1 — ML (정형 센서 → 작물 22종 분류) · 난이도 ⭐
- ✅ 1-1 환경 세팅 + 데이터 확보
- ✅ 1-2 데이터 로드 + 첫 탐색 (`explore.py`)
- ✅ 1-3 EDA 시각화 (`eda_plots.py`)
- ✅ 1-4 전처리 (`preprocess.py`)
- ✅ 1-5 모델① LogisticRegression + 평가 (97.3%)
- ▶ **1-6 모델② RandomForest** (LogReg와 비교, rice↔jute 개선 확인)
- ⬜ 1-7 모델③ XGBoost + **모델 비교·베스트 선정**
- ⬜ 1-8 Streamlit 데모 배포 (`app/phase1_ml.py` — 환경값 입력 → 작물 추천)
- ⬜ 1-9 발표 5장 + **보고서(수행내역서 양식) 조립** → `phase1_ml.md` 완성
- 🏁 **Phase 1 끝 = Streamlit 데모 라이브 + 보고서**

### Phase 2 — DL (시계열 + 비전) · 난이도 ⭐⭐⭐⭐ ← 차별화 핵심
**2-a 시계열 (LSTM)**
- ⬜ 2-1 PyTorch 환경 + 시계열 데이터 확보(가상 시뮬 or 공개)
- ⬜ 2-2 시계열 전처리 (시퀀스 윈도우 만들기)
- ⬜ 2-3 LSTM 학습 + 평가 (향후 환경 추세 예측)

**2-b 비전 (CNN) — ⭐ ML로 불가능한 새 능력**
- ⬜ 2-4 PlantVillage 데이터 확보 + 이미지 EDA
- ⬜ 2-5 이미지 전처리 (torchvision transforms/DataLoader)
- ⬜ 2-6 CNN 전이학습(ResNet/EfficientNet) 학습 + 평가
- ⬜ 2-7 Streamlit 확장 (잎 사진 업로드 → 병 진단)
- ⬜ 2-8 발표 5장 + 보고서
- 🏁 **Phase 2 끝 = 사진 올리면 진단 + 모델 저장(.pt)**

### Phase 3 — LLM (통합 처방 + 알림) · 난이도 ⭐⭐⭐
- ⬜ 3-1 Claude API 연동 기초 (숫자·라벨 → 자연어 처방)
- ⬜ 3-2 재배지식 RAG 구축 (농사로 가이드 검색)
- ⬜ 3-3 통합 파이프라인 (ML/LSTM 예측 + CNN 진단 + RAG → LLM 처방)
- ⬜ 3-4 알림봇 연동 (텔레그램)
- ⬜ 3-5 Streamlit 대시보드 (환경·진단·처방 한 화면)
- ⬜ 3-6 발표 5장 + **최종 포트폴리오 정리**(README 허브 3데모 링크)
- 🏁 **Phase 3 끝 = 사진+센서 → 자연어 처방 + 알림 도착 = 프로젝트 완성**

> 📌 매 Phase 공통 마무리 2칸: **Streamlit 배포 → 발표/보고서**. 이게 "한 Phase 끝" 신호.

## 📅 Phase 2 — DL (시계열 + 비전) ⭐ 이 프로젝트의 차별화 핵심

> **왜 DL?** 두 가지 다른 능력을 추가한다.
> ① 환경은 "흐름"이 중요 → **LSTM이 순서를 기억**
> ② 잎 사진은 정형 ML로 **애초에 불가능** → **CNN이 이미지를 읽음** (= ML→DL 점프가 진짜)

### 2-a 시계열 (LSTM/GRU)
| 항목 | 내용 |
|---|---|
| 입력 | 지난 N시간 환경 시퀀스(온습도·CO₂ 등) |
| 출력 | 향후 환경 추세 (고온/저습 등 사전 경고) |
| 데이터 | 스마트팜코리아 시계열 / 공공데이터 / 가상 시뮬레이션 |

### 2-b 비전 (CNN) — 차별화 포인트
| 항목 | 내용 |
|---|---|
| 입력 | 잎 사진 |
| 출력 | 질병 분류(건강/탄저병/잎곰팡이 등) |
| 데이터 | **PlantVillage**(Kaggle, 즉시) / AI Hub 병해충(한국) |
| 모델 | CNN + **전이학습**(ResNet/EfficientNet) |

- 🛠️ 새 라이브러리: **PyTorch + torchvision** (또는 TensorFlow/Keras)
- 포인트: ML 대비 "이미지 진단"이라는 **새 능력** 자체가 차별화 (정확도 비교 아님)

**📊 발표 + 🚀 배포 (Streamlit)**
| | 내용 |
|---|---|
| 📊 발표자료 | 학습곡선 + 혼동행렬 + 예측 샘플 + "ML이 못 하던 걸 DL이 함" |
| 🚀 배포 | **Streamlit 확장** — Phase 1 앱에 **사진 업로드 → 병 진단** 추가 |
| 산출물 | 코드 + 모델 저장(.pt) + 발표 5장 + 업그레이드된 앱 |
| 완료 기준 | Streamlit에서 잎 사진 올리면 진단 결과 표시 |

## 📅 Phase 3 — LLM (통합 재배 도우미) ⭐

> **핵심:** LLM은 병을 "진단"하지 않는다. Phase 1·2가 예측·진단한 걸 받아 **원인·조치를 사람 말로 처방**하고 **알림**을 보낸다.

### 들어가는 재료 3가지
| 재료 | 출처 | 예시 |
|---|---|---|
| 🔬 병 진단 | Phase 2 CNN | 잎곰팡이병 87% |
| 🌡️ 환경 예측 | Phase 1·2 (ML/LSTM) | 2시간 뒤 32도, 토양수분 30% |
| 📖 재배 지식 | RAG(농사로) | 작물별 재배·방제 가이드 |

### LLM이 만드는 처방 로직
| 조건 | 처방 문구 |
|---|---|
| 토양수분 낮음 | 💧 관수 필요 — 물 주세요 |
| 고온 예측 | 🌡️ 곧 고온, 환기·차광 준비 |
| 병 진단 양성 + 고습 | 🍃 통풍 늘리고 감염 잎 제거 |
| 질병 분류 양성 | 🔬 OO병 의심 — 약제·격리 안내 |
| CO₂/일사 부족 | ☀️ 보광 또는 환기 조정 |

### 동작 흐름
```
1. 센서 데이터 + 잎 사진 입력 (수동 업로드 or 스케줄)
2. ML/LSTM 호출 → 환경 예측
3. CNN 호출 → 병 진단
4. LLM에 전달 + RAG 재배가이드 → "이 상황 처방 써줘"
5. LLM이 자연어 처방 생성 → 화면 표시 / 이상 시 알림 📱
```

| 항목 | 내용 |
|---|---|
| **LLM 역할** | 숫자·라벨 → 자연스러운 한국어 처방으로 변환 |
| **알림 수단** | 텔레그램 봇 / Discord 웹훅 (쉬운 것부터) |
| **배우는 것** | LLM API(프롬프트·function calling), RAG, 알림 연동 |

- 🛠️ 새 기술: **Claude API** + **RAG** + **알림 봇**

**📊 발표 + 🚀 배포 (Streamlit + 실서비스)**
| | 내용 |
|---|---|
| 📊 발표자료 | 아키텍처 다이어그램 + 실제 처방 데모 영상/스크린샷 |
| 🚀 배포 | **Streamlit 대시보드**(환경·진단·처방 한 화면) + **알림봇 실서비스** |
| 산출물 | 코드 + 발표 5장 + Streamlit 대시보드 + 작동하는 알림봇 |
| 완료 기준 | 사진+센서 넣으면 자연어 처방이 나오고 이상 시 알림 도착 |

> 💡 최종 배포 옵션: Streamlit Community Cloud(무료·가장 쉬움) 또는 OCI 서버(oci-arm2, 자동배포 세팅됨 — 그때 상태 확인)

## 🆚 출근비서 대비 우위 (왜 갈아탔나)

| | 출근비서 | 스마트팜 |
|---|---|---|
| ML→DL 점프 | 시계열→시계열 (양적, 약함) | 정형→이미지 (질적, 강함) |
| DL 명분 | "XGBoost가 더 나을 수도" 리스크 | 비전은 ML로 불가능 → 명분 확실 |
| 컨셉 충돌 | 팀원과 겹침 | 없음 |
| 포트폴리오 | 단일 모달 시계열 | **멀티모달(정형+비전+언어)** |
